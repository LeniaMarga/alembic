Alembic Layout

StackLayout places elements vertically and adds whitespace between them. Set space to control the amount of whitespace or remove it. You can nest multiple StackLayouts with different spacing.

BoxLayout is a primitive to show a container of other elements. It has a uniform padding and borderWidth to configure the box. It can be inverted with invert to swap the text and background colour.

CenterLayout ensures a block-level element is placed in the horizontal middle of its parent. The element will be at most max width and spacing will be added at either side to ensure it. Use gutters to apply a padding to either side of the cantered element, outside of the max width. Use the intrinsic prop to use the centered elements width, while it is less that max.

ClusterLayout groups multiple items together based on their intrinsic width and separated by a common gutter that decided by the. Use the justify and align props to control the Flexbox justification and alignment. space prop.

SidebarLayout places two elements side-by-side if space permits it or stacks them on top of each other if not. The sidebar can either be a set width (using sideWidth) or use its intrinsic width (the width of the content inside it). The content element will fill up the rest of the horizontal space next to the sidebar.

Use the side prop to tell SidebarLayout which side to put the sidebar on, either "left" or right" sideWidth to specify a width for the sidebar. When the content's width is less than the contentMin percentage, the sidebar and content stack vertically.

SwitcherLayout dynamically displays a set of columns horizontally if there is enough space or stacks them vertically if not. It is based on threshold which is the minimum width that it will show elements horizontally. It has a limit where if exceeded elements will always display vertically.

There is a limit prop that says how many elements it is allowed to display horizontally, if exceeded all elements will display vertically. If filled dynamically, this limit should be taken into consideration, e.g. stop adding items after it has been reached.

CoverLayout fills a minimum vertical height and centers a principle element vertically in the layout. It also places accessory elements above/below which keep their intrinsic height. By default a nested "h1" element is the principle element, set centered to a CSS selector for an alternate principle element.

Use space to control the minimum space between all elements and set noPad if you don't want padding on the top-level layout itself. The default minimum height of the layout is "100vh" but it can be overidden with the minHeight prop.

GridLayout creates a responsive CSS Grid where each element has a min(imum) width. Element start at the minimum width and grow proportionally until there is space for "ghost" elements at which points all elements start from the minimum width again. Set the space prop to configure the gutter between elements.

GridLayout is suited for lists of same-sized content that are to be displayed at roughly the same size. For example, a set of cards showing people on a website. It avoids the last-row stretching of some Flexbox implementations.

FrameLayout ensures it's child element has a fixed aspect ratio by cropping around the center of the image. If content is provided instead of an image, that will be centered with Flexbox too. Set ratio to the desired aspect ratio in the format "16:9".

ReelLayout places elements horizontally with a set space between them and ensures overflow scrolling happens. Unless noBar is set, the ReelLayout will show scrollbars to indicate that there is horizontally scrollable content.

Use the height prop to force all laid-out elements to be a set height, otherwise it will use the elements' intrinsic height (within a Flexbox context). Set itemWidth to enforce a width on all elements instead of using their intrinsic width.

ImposterLayout positions an element over other elements. The positioned element is vertically and horizontally centered in it's nearest positioned parent and will not exceed it's height or width, unless breakout is set. It can optionally be fixed-d to the viewport instead. The width of the element is 50% of the positioned parent unless overridden in CSS, it is setup to not exceed 100% of the parent's height or width.

IconLayout is for placing icons before or after text. Use space to have a fixed gap between the text and icon, otherwise the natural whitespace from the HTML is used. Set label to markup an icon-only element with the correct accessibility attributes.
